Security Technology

Review Questions for Chapters 6 & 7

Chapter 6: Firewalls & VPNs

1. What is the typical relationship among the untrusted network, the firewall, and the trusted network?

The firewall sits as a controlled gateway between the untrusted network (typically the public Internet) and the trusted network (the organization's internal network). All traffic must pass through the firewall, which enforces an access control policy.

2. What is the relationship between a TCP packet and UDP packet? Will any specific transaction usually involve both types of packets?

Both are core transport layer protocols. TCP is connection-oriented and reliable, while UDP is connectionless and faster but doesn't guarantee delivery. A single transaction typically uses one or the other, not both.

3. How is an application layer proxy firewall different from a packet-filtering firewall?

A packet-filtering firewall operates at the network and transport layers, making decisions based on headers. An application layer proxy firewall operates at the application layer, acting as an intermediary and inspecting the actual content of traffic.

4. How is static filtering different from dynamic filtering of packets? Which is perceived to offer improved security?

Static filtering uses pre-configured, fixed rules, while dynamic filtering can create rules on the fly in response to traffic. Dynamic filtering is perceived as more secure because it's more adaptive.

5. What is stateful inspection? How is state information maintained during a network connection or transaction?

Stateful inspection monitors the state of active connections and makes decisions based on context. It maintains a state table that tracks all current connections, including details like IPs, ports, and sequence numbers.

6. Explain the conceptual approach that should guide the creation of firewall rule sets.

The guiding principle is "default-deny": "That which is not expressly permitted, is denied." The final rule should be an explicit "deny all," with rules ordered from most specific to most general.

7. What special function does a cache server perform? Why is this useful for larger organizations?

A cache server stores frequently requested web content locally. This reduces bandwidth usage, improves user experience, and can provide content filtering and logging.

8. Describe how the various types of firewalls interact with network traffic at various levels of the OSI model.

Packet filtering operates at Layers 3 & 4; stateful inspection at Layers 3 & 4 with state awareness; application proxy at Layer 7; circuit-level proxy at Layer 5.

9. What is a hybrid firewall?

A hybrid firewall combines several firewall techniques into a single system. Most modern firewalls are hybrids, incorporating packet filtering, stateful inspection, and application-level proxies.

10. Describe Unified Threat Management. Why might it be a better approach than single-point solutions? How does UTM differ from Next Generation Firewalls?

UTM is a single appliance combining multiple security functions (firewall, IDPS, antivirus, etc.). It offers simplified management and lower cost. UTM is a "jack-of-all-trades" for smaller businesses, while NGFW focuses on deeper inspection for enterprises.

11. What is a Next Generation Firewall (NextGen or NGFW)?

A NGFW is a deep-packet inspection firewall that adds application-level inspection, intrusion prevention, and user identity awareness to standard stateful inspection.

12. What is the primary value of a firewall?

To enforce an organization's access control policy at the network boundary, controlling what traffic is allowed to enter or leave the trusted network.

13. What is Port Address Translation (PAT) and how does it work?

PAT allows multiple devices on a LAN to be mapped to a single public IP address by using unique source port numbers for each connection. The firewall keeps a translation table to route returning traffic correctly.

14. How do screened host architectures for firewalls differ from screened subnet firewall architectures? Which offers more security?

Screened host uses a single firewall with a bastion host inside the trusted network. Screened subnet uses multiple firewalls to create a DMZ. Screened subnet offers more security due to multiple layers of defense.

15. What is a sacrificial host? What is a bastion host?

A sacrificial host is fully exposed with the assumption it may be compromised. A bastion host is heavily fortified to resist attack. In practice, the terms are often used interchangeably.

16. What is a DMZ? Is this really an appropriate name for the technology?

A DMZ is a neutral, controlled buffer network between trusted and untrusted networks. The name is appropriate as it describes a "buffer zone" where opposing forces can interact under controlled conditions.

17. What questions must be addressed when selecting a firewall for a specific organization?

Key questions include: required technology type, cost, ease of management, security policies, performance requirements, and whether to bundle with other services.

18. What is RADIUS? What advantage does it have over TACACS?

RADIUS is a protocol for centralized authentication, authorization, and accounting. Its advantage is widespread support across network equipment and software.

19. What is a content filter? Where is it placed in the network to gain the best result?

A content filter restricts access to certain online content. It's best placed at the network egress point to filter all traffic for all users.

20. What is a VPN? Why is it becoming more widely used?

A VPN is a secure, encrypted tunnel over a public network. It's widely used for secure remote access, cost-effective branch office connections, and protecting data on untrusted networks.

Chapter 7: IDPS & Scanning

1. What common security system is an IDPS most like? In what ways are these systems similar?

An IDPS is most like a burglar alarm. Both monitor for unauthorized activity and trigger alerts to notify responsible parties.

2. How does a false positive alarm differ from a false negative alarm? From a security perspective, which is less desirable?

A false positive incorrectly flags normal activity as an attack. A false negative misses an actual attack. False negatives are less desirable as they leave the organization unaware of real threats.

3. How does a network-based IDPS differ from a host-based IDPS?

A NIDPS monitors network traffic across segments, while a HIDPS monitors activity on a single host like system logs and file access.

4. How does a signature-based IDPS differ from a behavior-based IDPS?

Signature-based uses known attack patterns, while behavior-based establishes a baseline of normal behavior and flags deviations.

5. What is a monitoring (or SPAN) port? What is it used for?

A SPAN port receives copies of all network traffic passing through a switch. It's used to connect monitoring tools like NIDPS or packet sniffers.

6. List and describe the three control strategies proposed for IDPSs.

1. Control Centralization: All functions managed from a single location. 2. Control Fully Distributed: Functions applied at each monitored component. 3. Partially Distributed: Hybrid approach with local agents and central management.

7. What is a honeypot? How is it different from a honeynet?

A honeypot is a single decoy system to study attackers. A honeynet is a network of multiple honeypots creating a more realistic decoy environment.

8. How does a padded cell system differ from a honeypot?

A honeypot lures attackers into a decoy. A padded cell is a quarantined environment that a live system is moved into after an attack is detected.

9. What is network footprinting?

The first phase of reconnaissance where an attacker gathers publicly available information about a target to define its attack surface.

10. What is network fingerprinting?

Active reconnaissance involving probing a target's network to gather detailed information about systems, OS, open ports, and services.

11. How are network footprinting and network fingerprinting related?

They are sequential steps in reconnaissance. Footprinting provides the broad map, while fingerprinting fills in specific technical details.

12. Why do many organizations ban port scanning activities on their internal networks?

To detect malicious insiders, reduce network noise, and enforce policy that such activities must be authorized.

13. Why would ISPs ban outbound port scanning by their customers?

Outbound scanning often indicates a compromised system being used to attack other networks, which ISPs want to prevent.

14. What is an open port? Why is it important to limit the number of open ports to those that are absolutely essential?

An open port accepts incoming connections. Each represents a potential entry point for attackers, so limiting them reduces the attack surface.

15. What is a system's attack surface? Why should it be minimized when possible?

The attack surface is the sum of all potential points where unauthorized users can try to enter or extract data. Minimizing it reduces opportunities for attackers.

16. What is a vulnerability scanner? How is it used to improve security?

An automated tool that scans systems to identify known security weaknesses. It's used proactively to find and patch vulnerabilities before attackers exploit them.

17. What is the difference between active and passive vulnerability scanners?

Active scanners probe targets directly, while passive scanners monitor network traffic without direct interaction. Active is more accurate but intrusive; passive is stealthier but less comprehensive.

18. What is Metasploit Framework? Why is it considered riskier to use than other vulnerability scanning tools?

A penetration testing tool that provides vulnerability information and aids in exploit development. It's riskier because it actively exploits vulnerabilities, which can crash systems.

19. What kind of data and information can be found using a packet sniffer?

Unencrypted usernames/passwords, emails, web pages visited, files transferred, and network protocol information.

20. What capabilities should a wireless security toolkit include?

A wireless adapter with monitor mode, network discovery software, traffic analysis tools, and tools for assessing encryption strength.
